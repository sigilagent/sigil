The idea

Most agents evolve fuzzily. Sigil evolves by construction.

The first time Sigil meets a class of task, a frontier model authors an AG-IR — a typed graph contract — which a compiler lowers into a runnable object-spatial procedure. That procedure is persisted on the graph and, from then on, executed by a cheap model. The expensive model is paid once, to build the harness. The cheap model rides it forever.

04 · RUN

Execute

It runs the skill it built — on the small, cheap model.

05 · LEARN

Distill

Outcome + durable facts persist as memory. Result returned.

HITReuse the crystallized procedure on the small model — no frontier call.
MISS / PARTIALFrontier crystallizes a new procedure, or mutates an existing one into a new version.
VALVEOn small-model failure, the frontier mutates & retries — evolution under pressure.

Why it matters: the crystallized procedure carries structure a weak model cannot skip — step order, embodied tools, grounded verification. The cheap model isn't asked to be smart; it's asked to follow a smart procedure. That is where capability transfers from the frontier model to the cheap one.

Everything lives on the graph

Your whole agent is one living graph — with nothing on disk.

Nothing to author, nothing to configure. Its identity, skills, memory, connections and schedules all live together as one connected whole — and change the moment you ask.

Procedural

Skills

The reusable skills it has built — each one covers a whole kind of task, not just one. This is what it has learned to do.

Episodic

History

Every run, its outcome and a summary. The record of what the agent has done and how it went.

Semantic

Memory

Durable facts, retrieved by deterministic lexical recall and injected at run time — grown automatically by distilling each completed task.

Soul is identity and config as state you mutate in place — persona, ethos, model tiers, channels. awaken() rebinds the cognition from the graph on every run, the way a compiled OSP agent rebinds its llm in run(). The crystallized procedure stays class-general; instance-specific memory is injected only at execution time, never baked in.

Skills as plugins

Hand it a skill written in plain Markdown, a ready-made recipe, or one it built earlier — and it just works.

Bring your own — no restart

Add tools and skills whenever. They work on your next command.

Give it new abilities on the fly — connect any tool server, or hand it a ready-made skill — and it can use them right away. Nothing to restart, no files to edit.

give it new tools
# connect any tool server (an MCP server)
sigil add-mcp github stdio gh-mcp
sigil add-mcp search http http://localhost:9000/mcp

# see the servers and the tools they bring
sigil mcp
# it can use those tools from your very next command
give it new skills
# hand it a skill written in plain Markdown — it learns to run it
sigil register-skill ./my-skill/SKILL.md

# or a ready-made recipe (no model needed)
sigil register-skill ./procedure.agir agir

# or drop in one it already built
sigil register-skill ./skills/report_v1.jac osp
Download & install

One command. The whole agent.

The installer pulls the self-contained jac runtime, fetches Sigil, provisions its one compiler dependency, and drops a sigil launcher on your PATH. No root, no pip install, safe to re-run.

$ 
curl -fsSL https://github.com/sigilagent/sigil/releases/latest/download/install.sh | bash

macOS & Linux · installs to ~/.sigil/app · override with SIGIL_HOME, SIGIL_BIN_DIR, SIGIL_REF

01

Native runtime

Installs one small self-contained program — no Python, no dependencies to manage.

02

Fetches Sigil

Clones the source into ~/.sigil/app (or a tarball if you have no git).

03

Provisions PyYAML

The one dependency the compiler needs and the binary doesn't bundle.

04

Puts sigil on PATH

A launcher so sigil serve / sigil solve work from anywhere.

Prefer to install by hand?
manual install
# 1 · install one small self-contained runtime
curl -fsSL https://raw.githubusercontent.com/jaseci-labs/jaseci/main/scripts/install.sh | bash
export PATH="$HOME/.local/bin:$PATH"

# 2 · clone Sigil & provision the compiler dependency
git clone https://github.com/sigilagent/sigil.git && cd sigil
jac install pyyaml

# 3 · configure cognition (frontier needs a provider key)
export OPENAI_API_KEY=sk-...
export SIGIL_FRONTIER=gpt-5                 # default
export SIGIL_SMALL=ollama_chat/qwen3:32b     # can be local

Use it

Just tell it what you want — in plain English.

Point it at models once — a hosted one like gpt-4o for the hard thinking, and a small or fully-local one (via ollama) for the everyday runs. After that, you just talk to it.

your terminal
# ask it to do something — it works out HOW, just once
sigil solve "turn report.pdf into a clean CSV"

# ask for something similar — it reuses what it learned (fast & cheap)
sigil solve "pull the tables out of invoice.pdf"

# tell it your preferences — they stick, no config files to edit
sigil teach "always give me CSV with a header row"

# see what it is, and everything it's done
sigil soul
sigil tasks
